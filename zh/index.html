01 Overview

很多选择真正消耗人的地方，并不是选项有多复杂，而是它们都差不多。

WhichOne 把这种小小的犹豫变成一个轻松的决策流程：选择场景，让黑猫带着用户进入玩法，揭晓结果，然后继续往前走。

它面向的是低风险的日常选择：吃什么、去哪儿、做什么、谁先来、怎么分组。目标不是替用户做重大判断，而是让那些每天都会卡住人的小选择更快、更有趣地完成。

Decision Companion

处理那些很小、但真的会卡住人的选择。

当每个答案都“也可以”时，WhichOne 给用户一个明确的下一步，而不是继续停在空白里。

02 黑猫角色
黑猫 IP

用一个角色贯穿整个决策体验。

黑猫贯穿 WhichOne 的主题、内置选项、小游戏、分组、卡牌、硬币和结果时刻。它让应用不像一组松散工具，而更像一个陪用户做决定的小伙伴。

它有点调皮的表情会缓解用户纠结时的僵住感。应用传达的感觉是：别想太久，让黑猫帮你先选一个。

03 Options
Built-in Options

从吃什么、去哪儿、做什么这些高频问题开始。

WhichOne 为餐食、地点和活动准备了内置选项。用户可以根据偏好快速生成候选项，也可以在开始游戏前自由编辑、删除或补充选项。

Option Set 01

吃什么。

餐食选项帮助用户跳过最常见的日常纠结。应用可以先给出一个可用列表，用户再按口味添加、删除或修改。

  • 适合晚饭、外卖、聚餐等高频场景。
  • 选项可编辑，让结果更贴合真实偏好。
Option Set 02

去哪儿。

当计划还没有目的地时，应用先给一个方向：附近地点、短途出门、周末安排或旅行候选，都可以成为决策选项。

  • 适合周边计划、旅行和周末出门。
  • 结果给出方向，最后决定权仍在用户手里。
Option Set 03

做什么。

休闲、家务、家庭互动和个人待办都可以变成一次轻松抽签。生成列表带来行动起点，编辑能力保留个人偏好。

  • 适合空闲时间、家庭活动和小任务。
  • 让“都行”变成一个可以执行的小结果。
04Modes
Solo & Party

单人模式和派对模式覆盖两类决策场景。

单人模式用于一个人自己的小选择：选择主题、编辑选项、揭晓结果，并保留历史方便再次游玩。

派对模式把决策变成多人共享的现场：可以抽一个人、围着一台设备一起玩，也可以录入参与者后进行分组。

05Games
Game Library

一组足够精致的决策小游戏。

WhichOne 不是把同一个随机器换几套名字。转盘、卡牌、扭蛋、硬币、答案之书、骰子和派对工具会给选择不同的节奏，让反复使用也保持新鲜。

不只是随机

游戏感让结果更容易被接受。

应用不会立刻丢出一个答案，而是让用户看到转盘减速、卡牌翻开、扭蛋抽取、刮开结果或等待一次小揭晓。

01

玩法多，重复感少

不同的游戏形式让高频决策不总像按同一个按钮。

02

手势也是答案的一部分

转、刮、翻、点、多人触摸，都让决策过程更有参与感。

03

短暂等待让结果更有分量

期待感会让随机答案更像一次小事件，尤其适合派对场景。

06Care
Visual & Accessibility

明亮色系、清晰反馈和实用的辅助体验。

WhichOne 使用高明度黄色、强黑色对比、活泼辅助色、震动反馈和大字体支持，让体验保持清楚、可点、可感知。

温暖黄色与黑色对比

黄色让应用轻快，黑色承担文字、按钮和黑猫识别。辅助色用于场景和结果反馈，但不会只靠颜色传递关键信息。

震动反馈

操作确认、结果出现和状态变化会配合 Apple 设备触感反馈，让关键瞬间更容易被感知。

2 倍字号与大字体

关键标题、选项、按钮和结果反馈都考虑更大字体显示。即使系统字体放大到 2 倍，核心流程仍尽量保持可读、可点、可完成。

多语言与设备布局

覆盖简体中文、繁体中文、日语、韩语和英语，也考虑 iPhone、iPad、横屏、竖屏下的单人和多人使用。

Product Notes

一些边界说明。

WhichOne 的定位是日常轻量决策和多人互动，不用于高风险专业判断。

用途边界：适合吃什么、去哪儿、做什么、谁先来、如何分组等低风险场景；不用于金融、医疗、法律等专业决策。

随机与娱乐：应用不承诺预测未来，不包含真钱下注、现金奖励或博彩机制。

数据与内购：用户选项、最近选择和历史记录主要保存在设备本地；可选内购用于移除广告。
